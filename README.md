CT-FIAS: Clinical Trial Financial Impact Analysis System 🧬📊

CT-FIAS is a production-grade, multi-agent AI pipeline designed to filter biotech and pharmaceutical market noise. By utilizing a local LLM framework, it cross-references daily press releases against empirical regulatory science to separate "hype" from "fact," outputting verified, human-readable financial reports.

**Note:** This pipeline was autonomously engineered from a raw Product Requirements Document (PRD) using a custom LangGraph meta-engineering factory.

## 🚀 Key Features

* **100% Local & Air-Gapped:** Zero financial data or prompts leave the host machine. Powered entirely by Gemma 4 (31B) running locally via Ollama.
* **Dual-Agent Reasoning Chain:** Utilizes CrewAI with an explicit context chain (`context=[fda_task]`) between an FDA Regulatory Expert agent and a Financial Analyst agent.
* **Defensive AI Engineering:** Built with strict anti-hallucination guardrails, length verification, and forced `source_url` injection for human auditing.
* **Resilient Orchestration:** Managed by Dagster using a Pipe-and-Filter architecture. Handles VRAM exhaustion by triaging news relevance and enforcing sequential processing.

## 🧠 The LangGraph Meta-Engineering Factory
Before analyzing a single stock, the codebase itself was generated by an AI software factory:
1.  **The Architect (Human-in-the-Loop):** A LangGraph agent reads the PRD, pauses to ask clarifying constraint questions via the terminal, and generates a strict `ARCHITECTURE.md`.
2.  **Team A (The Builders):** An engineering subgraph plans files, writes business logic, and performs static reviews.
3.  **Team B (DevOps & Micro-Testing):** A DevOps subgraph physically executes the generated Python scripts via `subprocess.run()`. It captures real terminal tracebacks (`stderr`) and feeds them back to the LLM with a `fix_history` state to break hallucination loops until the code executes perfectly.

## 💻 Hardware & Requirements

This system is optimized for high-performance edge hardware to handle the 31-billion parameter model.
* **Target Hardware:** msi EdgeXpert AI Mini Desktop (NVIDIA GB10 Grace Blackwell) or equivalent.
* **Memory:** 128GB LPDDR5 Unified Memory recommended to prevent OOM errors.
* **OS:** Ubuntu 24.04.4 
* **Dependencies:** `Ollama`, `CrewAI`, `LangGraph`, `Dagster`, `yfinance`, `sqlite3`, `fpdf`

## 🛠️ Architecture

* **Ingestion:** Daily cron job via `yfinance` → SQLite (with strict `UNIQUE` deduplication).
* **Agentic Core:** Sequential processing via CrewAI (Gemma 4).
* **Delivery:** Markdown to PDF generation via `fpdf`.

Workflow step by step:
1. Git clone the code to your local directory 

2. Set up local python environment
python -m venv .venv
source .venv/bin/activate 

3. Install any missing python packages 
pip install -r requirements.txt

4. python software_architect_langgraph.py --design PRD.md

5. python3 software_team_langgraph.py --design "ARCHITECTURE.md" --task "Phase 1: Ingestion (step1_data.py)

[ ] Task 1.0: Write the pytest suite for database setup and duplicate-prevention logic.

[ ] Task 1.1: Implement setup_database executing the strict SQLite schema.

[ ] Task 1.2: Implement ingest_daily_news using yfinance. Ensure ticker aliases are handled and exceptions do not crash the loop."

6. cd deployed_pipeline, make a file named tickers.txt, write GILD, BIID stock tickers into the file.

7. "dagster dev -f dagster_pipeline.py" start the pipeline

8. "ollama launch claude --model gemma4:31b" use claude code to debug if has error message.
